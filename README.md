# <H1> ethanbouterey.github.io <P> </H1>
<H2> I'm Ethan Bouterey, an Information Systems graduate from Victoria University, based in New Zealand. I build with AI as the accelerant - tools I would actually want to use myself, not demos for their own sake.<P></P></H2>
<P><B>This repo hosts my portfolio. Each project is a single-file HTML/CSS/JavaScript app, no build step, deployed straight to GitHub Pages.</B></P>
<P><B>What's Here:</B></P>
<B>OpsDesk</B> - a working model of an IT service desk for a Microsoft 365 environment. ITIL ticket triage with SLA tracking, identity and MFA management in an Entra-style console, Intune-style device compliance, and a documented knowledge base. Built to understand how the pieces of a modern cloud service desk fit together. It is a learning project, not a live system I have administered, and I'm upfront about that.</P>
<P><B>Launchpad</B> - an AI career hub. Live NZ job search through the Adzuna API, AI-scored job matching, CV tailoring, and an AI mock interview. Single-file app with no backend.</P>
<P><B>LeadFlow</B> - a B2B lead generation and outreach tool. Business search, a lightweight CRM with follow-up tracking, AI-written outreach, and pipeline analytics.</P>
<P><B>Personal site</B> - the landing page that ties it together.</P>
<P><B>How it was built:</B></P>
<P>Hand-written HTML, CSS, and vanilla JavaScript. No frameworks, no build pipeline, no dependencies. AI tools (Claude, mostly) sped up the writing, but the architecture, the design decisions, and the debugging are mine. The point was to learn the systems thinking behind each tool, not to glue a library together.
